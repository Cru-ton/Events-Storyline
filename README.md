# Events Storyline

## Introduction
The Events Storyline project is designed to provide a comprehensive and interactive platform for managing and showcasing various events. This documentation serves to guide developers and users in understanding the project's structure, features, and usage.

## Features
- **User Management**: Admins can manage users and permissions effectively.
- **Event Scheduling**: Create, update, and delete events while managing their schedules.
- **Notifications**: Send notifications for upcoming events and updates.
- **Interactive Calendar**: A user-friendly calendar interface to visualize events.
- **Custom Themes**: Support for customizing the look and feel of the event interface.

## Installation Instructions
1. **Clone the Repository**:  
   ```bash
   git clone https://github.com/Cru-ton/Events-Storyline.git
   cd Events-Storyline
   ```  
2. **Install Dependencies**:  
   ```bash
   npm install
   ```  
3. **Start the Application**:  
   ```bash
   npm start
   ```  
4. **Access the App**:  
   Open your browser and navigate to `http://localhost:3000`

## Usage
- **Creating an Event**:  
   Navigate to the events section and click on 'Create Event'. Fill in the required details and save.
- **Viewing Events**:  
   All upcoming events will be listed in the calendar and event section.
- **Editing Events**:  
   Select an event from the list to view details and choose the edit option.

## Customization
- Customize themes by modifying the CSS files located in the `styles` directory.
- Update configurations in the `config.js` file for personalized settings.

## Technologies
- **Frontend**: React, Redux, CSS
- **Backend**: Node.js, Express
- **Database**: MongoDB
- **Deployment**: Heroku
